# Desafio_Mit-mi

The Bridge School Bootcamp DS — Desafío 5 · Mitümi (organización de eventos).

## Estructura del repositorio

| Carpeta | Contenido |
|---|---|
| `docs/analisis/` | Análisis de mercado, invitados/presupuestos y stakeholders (HTML) |
| `docs/material-cliente/` | Documentación recibida de Mitümi (briefing, directorio de espacios, ficha de ponente, presentación) |
| `docs/equipo/` | Acuerdos y organización interna del equipo |
| `docs/presentaciones/` | Presentaciones del desafío y material de formación |
| `api/` | Contrato de la API de eventos (`contrato_api_eventos_2.md`) |
| `api/mock/` | API de prueba en Python (ver su `README.md` para ejecutarla) |
| `diseno/wireframes/` | Wireframes de la aplicación (HTML) |
| `grupos/grupo2-presupuestos-servicios/` | Entregables del Grupo 2: presupuestos y servicios |
| `grupos/grupo3-ponentes/` | Entregables del Grupo 3: ponentes (BBDD, agentes IA, interfaz) |

El backend del proyecto (`proyectoTripulacionesBackend`) vive en su propio repositorio, fuera de este.
